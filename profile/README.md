## 🔍 Overview

<a href="#exportForiTunes">Export for iTunes</a> provides the library export and backup capabilities that Apple Music and iTunes don't include natively — exporting playlists in standard formats, copying music files with organizational structure preserved, and producing backups of library data that protect against library corruption or migration needs.

The <a href="#exportForiTunes">playlist export to standard formats</a> converts Apple Music and iTunes playlists to M3U, XSPF, and other universal playlist formats — creating playlist files that music players other than Apple Music can read. The native Apple Music playlist format is proprietary; Export for iTunes produces the open playlist formats that VLC, foobar2000, AIMP, and other music software use for importing playlists. The <a href="#exportForiTunes">music file copying with structure</a> copies the actual audio files from the iTunes/Music library to a destination folder while preserving the artist/album/track organizational hierarchy — creating a portable copy of the music collection that maintains the organizational structure the library uses. This copy capability serves backups, transfers to other devices and players, and situations where the original library location changes.

The <a href="#exportForiTunes">library data CSV export</a> produces a spreadsheet-compatible export of the music library's metadata — track titles, artists, albums, genres, play counts, ratings, dates added, and other library metadata fields exported to CSV for external analysis, documentation, or import into other music management systems. The <a href="#exportForiTunes">metadata preservation</a> maintains the complete ID3 tag information — embedded metadata in the audio files — when copying tracks, ensuring that exported music retains the correct artist, album, track number, year, and other tag data in the destination copy.

<a href="#exportForiTunes">Selective playlist and track export</a> exports specific playlists or selected tracks rather than the complete library — exporting only the content relevant to a specific use case rather than always handling the complete collection. <a href="#exportForiTunes">Folder structure configuration</a> specifies how exported files organize — artist/album/track hierarchy, flat folder with renamed files, or other organizational schemes matching the destination's expectations.

<a href="#exportForiTunes">Large library handling</a> processes music libraries of thousands of tracks without performance issues — export operations on complete collections that take time to process complete reliably in the background. <a href="#exportForiTunes">One-time purchase from the Mac App Store</a> provides permanent Export for iTunes access.
